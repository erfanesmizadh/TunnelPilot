## 🔐 انواع تانل

### 🔐 تانل وایرگارد
VPN سریع در سطح kernel با رمزنگاری مدرن.

**مزایا:**
- ⚡ سریع‌ترین گزینه (تأخیر کمتر از یک میلی‌ثانیه)
- 💻 کم CPU/حافظه
- 🔒 رمزنگاری مدرن (Curve25519)
- 📦 کد کمینه (۶۰۰ خط)

**معایب:**
- دیدپذیری سطح شبکه
- ممکن است توسط DPI پیشرفته تشخیص داده شود

**تنظیم:**
```bash
منو → ۱
نام تانل: wg_bypass
IP عمومی همتا: <ip>
نقش سرور: ۲ (ایران)
```

**اتصال مشتری:**
```bash
# تولید کلیدها
wg genkey | tee privatekey | wg pubkey > publickey

# ایجاد پیکربندی
[Interface]
Address = 10.10.1.2/30
PrivateKey = <کلید>

[Peer]
PublicKey = <کلید عمومی سرور>
Endpoint = <IP سرور>:51820
AllowedIPs = 0.0.0.0/0

# اتصال
sudo wg-quick up <پیکربندی>
```

### 🔗 تانل SSH
تانل‌سازی مخفی‌شده پورت از طریق پروتکل SSH.

**مزایا:**
- 🔓 SSH در هر سروری فعال است
- 🎭 می‌تواند از هر پورتی استفاده کند (۸۰، ۴۴۳ و غیره)
- 🔒 رمزنگاری بومی
- 📱 در همه جا کار می‌کند

**معایب:**
- کمی کندتر از وایرگارد
- استفاده CPU بیشتر

**تنظیم:**
```bash
منو → ۲
نام تانل: ssh_bypass
نام‌کاربری SSH دور: نام‌کاربری
میزبان SSH دور: server.example.com
پورت SSH دور: ۲۲
پورت محلی: ۸۰۸۰
```

**استفاده مشتری:**
```bash
# پروکسی SOCKS5
SOCKS5: 127.0.0.1:8080

# با curl
curl -x socks5://127.0.0.1:8080 http://example.com

# با مرورگر
تنظیمات پروکسی → SOCKS5 → 127.0.0.1:8080
```

### ☁️ تانل کلودفلر
دسترسی شبکه بدون اعتماد از طریق لبه کلودفلر.

**مزایا:**
- 🛡️ حداکثر محافظت علیه فیلترینگ
- 🌍 CDN جهانی کلودفلر
- 🔒 حفاظت DDoS شامل
- 💯 تقریباً غیرممکن برای مسدود کردن

**معایب:**
- نیاز به حساب کلودفلر
- نیاز به دامنه
- تأخیر کمی بیشتر

**تنظیم:**
```bash
# ۱. ایجاد حساب کلودفلر
https://dash.cloudflare.com

# ۲. ایجاد تانل
Tunnels → ایجاد → Cloudflared

# ۳. کپی توکن اعتبار

# ۴. اجرای منو
منو → ۳
نام تانل: cf_bypass
چسباندن توکن از کلودفلر
```

## 🇮🇷 شکست فیلترینگ ایران

### سطح‌های فیلترینگ و راه‌حل‌ها

**سطح ۱: فیلترینگ سبک**
```
→ استفاده: وایرگارد (منو ۱)
→ دلیل: سریع‌ترین گزینه
```

**سطح ۲: فیلترینگ متوسط (ایرانسل/همراه)**
```
→ استفاده: تانل SSH (منو ۲)
→ پورت: ۴۴۳ یا ۸۰ برای مخفی‌سازی
```

**سطح ۳: تشخیص DPI سنگین**
```
→ استفاده: تانل کلودفلر (منو ۳)
→ دلیل: مسدود کردن تمام کلودفلر = مسدود کردن میلیون‌ها سایت
```

**سطح ۴: فیلترینگ بسیار سخت (ناشناس)**
```
→ استفاده: حالت ترکیبی (منو ۴)
→ هر ۳ تانل با هم کار می‌کند
→ چندین لایه بازتاب
```

### اجتناب پیشرفته از DPI

**برای SSH با مخفی‌سازی:**
```bash
ssh -N -D 8080 -p 443 \
    -o ProxyCommand="curl -x proxy.example.com:8080 -k https://%h:%p" \
    username@server.example.com
```

**برای وایرگارد بر روی پورت‌های مختلف:**
```bash
wg set wg_bypass listen-port 443
```

**برای کلودفلر با دامنه سفارشی:**
```
۱. رفتن به پیشخوان کلودفلر
۲. تنظیم دامنه سفارشی
۳. فعال‌سازی کش
۴. تنظیم محدودیت نرخ پرخطر
```

```
  حافظه:      < ۲۰ مگابایت

  Ping:       ۲۰-۵۰ میلی‌ثانیه
  نفوذپذیری: ۱۰۰-۳۰۰ Mbps

  Ping:       ۳۰-۱۰۰ میلی‌ثانیه

```

## 📞 پشتیبانی

### دریافت کمک

۱. **بررسی مستندات**: راهنماهای این مخزن را بخوانید
۲. **مشاهده گزارش‌ها**: `tail -f /var/log/tunnelpilot.log`
۳. **باز کردن Issue**: مسئله خود را توصیف کنید
۴. **بررسی FAQ**: ببینید QUICK_START.md

### پرسش‌های متداول

**س: کدام تانل را باید استفاده کنم؟**  
پ: برای سرعت → وایرگارد، برای فیلتر → SSH، برای مسدودیت سنگین → کلودفلر

**س: آیا استفاده از آن ایمن است؟**  
پ: بله، تقویت‌شده با تصحیح‌های امنیتی و گزارش‌دهی

**س: آیا می‌توانم چند تانل استفاده کنم؟**  
پ: بله! حالت ترکیبی (منو ۴) هر ۳ را با هم اجرا می‌کند

**س: آیا ISP من آن را تشخیص می‌دهد؟**  
پ: بستگی به قدرت ISP دارد. کلودفلر سخت‌ترین برای مسدود کردن است

**س: سرعت چقدر است؟**  
پ: وایرگارد تأخیر کمتر از یک میلی‌ثانیه دستیافت می‌کند، بسیار برای streaming بهترین است
